# 远程桌面连接客户端 - 用户手册

## 1. 简介

远程桌面连接客户端是一个轻量级的 Windows 应用程序，允许您通过 IP 地址和端口连接到远程计算机。该应用程序提供了一个简单的图形界面，用于管理远程桌面连接。

## 2. 功能特性

- 通过 IP 地址和端口连接远程桌面
- 支持自定义 RDP 端口（默认为 3389）
- 保存和管理多个远程连接配置
- 用户友好的图形界面
- 连接状态指示

## 3. 系统要求

- Windows 10/11 (64位)
- Go 1.19 或更高版本（用于编译源代码）
- 目标计算机启用远程桌面服务
- 网络连接

## 4. 安装

### 方法 1: 使用预编译的可执行文件

1. 下载 `RemoteDesktopClient.exe`
2. 将其保存到您的计算机上的任意位置
3. 双击运行应用程序

### 方法 2: 从源代码编译

1. 安装 Go 1.19 或更高版本
2. 下载源代码
3. 在项目目录中运行以下命令：
   ```
   go mod tidy
   go build -ldflags="-H windowsgui" -o RemoteDesktopClient.exe .
   ```

## 5. 使用指南

### 5.1 启动应用程序

双击 `RemoteDesktopClient.exe` 启动应用程序。您将看到一个包含连接列表和操作按钮的主窗口。

### 5.2 创建新的远程桌面连接

1. 点击"新建连接"按钮
2. 在弹出的对话框中填写以下信息：
   - 连接名称: 为连接指定一个名称
   - IP 地址: 目标计算机的 IP 地址
   - 端口号: RDP 服务的端口号（默认为 3389）
   - 用户名: 用于登录的用户名
   - 域: 如果适用，输入域名
   - 全屏模式: 选中以启用全屏模式
   - 宽度/高度: 设置连接窗口的尺寸
3. 点击"连接"按钮

### 5.3 连接到已保存的连接

1. 在连接列表中选择一个连接
2. 点击"连接"按钮

### 5.4 管理连接

- 使用"刷新列表"按钮更新连接列表
- 在新建连接对话框中创建更多连接

## 6. 故障排除

### 6.1 无法连接到远程计算机

- 确认 IP 地址和端口号正确
- 确认目标计算机已启用远程桌面服务
- 检查防火墙设置，确保 RDP 端口未被阻止
- 确认网络连接正常

### 6.2 连接后无响应或黑屏

- 检查网络连接质量
- 确认目标计算机处于活动状态
- 尝试降低分辨率设置

### 6.3 应用程序无法启动

- 确认您在 Windows 系统上运行此应用程序
- 确认系统满足最低要求

## 7. 安全注意事项

- 只连接到受信任的远程计算机
- 使用强密码保护您的账户
- 定期更新系统和应用程序
- 不要在不受信任的网络上使用远程桌面连接

## 8. 技术支持
